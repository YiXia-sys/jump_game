# 任务列表：竞技模式（回合制多人赛跑对战）

## 任务 1：Mulberry32 伪随机数生成器

- [ ] 在 `index.html` 中实现 `mulberry32(seed)` 函数，接受 32 位整数种子，返回一个每次调用产生 [0, 1) 浮点数的函数
- [ ] 实现 `seededRandom(prng, min, max)` 辅助函数，基于 PRNG 生成指定范围的整数
- [ ] 验证：相同种子产生完全相同的序列

> 需求：4.2, 4.3 | 属性：1

## 任务 2：确定性赛道生成

- [ ] 实现 `generateRacetrack(seed)` 函数，使用 Mulberry32 生成 20 个平台的赛道
- [ ] 平台参数（位置、宽度、形状、间距、方向）全部由 seeded PRNG 决定
- [ ] 第 1 个平台为起点，第 20 个为终点，均不设为 Mystery_Platform
- [ ] 第 2-19 个平台以 20% 概率标记为 Mystery_Platform
- [ ] 特殊平台（移动/缩小/限时）复用经典模式逻辑，互斥分配

> 需求：4.1-4.6, 7.1 | 属性：2

## 任务 3：游戏服务器基础框架 (`server.js`)

- [ ] 创建 `server.js`，使用 Node.js + `ws` 库启动 WebSocket 服务，默认端口 3000
- [ ] 实现消息路由：解析 JSON 消息，按 `type` 字段分发到对应处理函数
- [ ] 实现连接管理：为每个连接分配唯一玩家 ID，发送 `welcome` 消息
- [ ] 实现心跳检测：5 秒超时判定断线
- [ ] 格式错误消息返回 `error` 类型响应

> 需求：2.1-2.3, 2.6, 2.7 | 属性：4, 22

## 任务 4：房间管理器

- [ ] 实现 `RoomManager` 类：`createRoom(hostId)`、`joinRoom(roomCode, playerId, playerName)`、`removePlayer(playerId)`、`getRoom()`、`destroyRoom()`
- [ ] 房间码生成：4 位字母数字，确保唯一
- [ ] 房间人数限制 2-6 人
- [ ] 加入验证：房间不存在、已满、已开始 → 返回对应错误
- [ ] 新玩家加入时广播更新后的玩家列表
- [ ] Host 断线时自动转移给下一位玩家

> 需求：3.1-3.9 | 属性：3, 5, 20, 23

## 任务 5：回合调度器

- [ ] 实现 `TurnScheduler`：`startGame()`、`rollDice()`、`reportJumpResult()`、`endTurn()`、`skipTurn()`
- [ ] 游戏开始时：生成 Platform_Seed，确定 Turn_Order（按加入顺序），所有玩家 Platform_Index = 1
- [ ] 掷骰子：生成 1-6 随机整数，应用 Dice_Modifier（加值上限 12，最大值截断）
- [ ] 跳跃结果处理：成功 → Platform_Index + 1，失败 → 不变，Jump_Quota - 1
- [ ] Jump_Quota = 0 时结束回合，轮转到下一位非 Skip_Turn 玩家
- [ ] Skip_Turn 处理：自动跳过，skipTurns - 1
- [ ] 胜利判定：Platform_Index >= 20 → 设为 20，结束对局

> 需求：2.4, 2.5, 3.8, 6.1-6.14, 10.1-10.4 | 属性：6, 7, 8, 9, 10, 11, 12

## 任务 6：事件仲裁器

- [ ] 道具使用验证：持有检查、回合检查、蓄力状态检查

- [ ] Shield 抵消负面事件后消失
- [ ] Platform_Index 变更后 clamp 到 [1, 20]
